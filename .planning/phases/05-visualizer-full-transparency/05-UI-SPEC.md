# Phase 5 — UI Design Contract

> Visual and interaction contract for NSPanel ghost cursor, SwiftUI HUD, H.265 replay, 3D timeline, and session diff.
> Audience: Akeil (single-user, local-only, no WCAG/a11y required).
> Generated by gsd-ui-researcher from locked ROADMAP success criteria + STACK.md NSPanel section + ARCHITECTURE.md L7 recipe.

| Property | Value |

## Visual Architecture

### Overlay Type

| Property | Specification |
|----------|---|
| Window level | `.popUpMenu` (above all user windows except floating palettes) |
| Transparency | Fully transparent background (framebuffs RGBA); drawn elements only |
| Behavior | Ignores mouse/keyboard input (`ignoresMouseEvents=true`) |
| Spaces | Joins all Spaces (`canJoinAllSpaces=true`) |
| Activation | Non-activating (does not steal focus from target app) |
| Window style | `.borderless` (no title bar, chrome, or shadow) |
| SCContentFilter | **Excludes overlay window ID from all verifier captures** (macOS 15+ Tahoe mandatory, replaces `sharingType=.none` which is broken post-15) |

**Pitfall mitigation:** P9 (ScreenCaptureKit captures own overlay — use SCContentFilter), P10 (macOS 15+ sharingType=.none broken), P11 (WindowServer CPU spike — single CAShapeLayer per element, hide during verify windows).

## Ghost Cursor

### Rendering

| Property | Specification |
|----------|---|
| Implementation | **NSView.draw() override, NOT CALayer** (Pitfall P12 — documented WindowServer perf bug at >10 actions/sec with CALayer animation) |
| Shape | Circle with crosshair: 16px outer diameter, 1px stroke (SF Symbols `crosshair.ova` fallback) |
| Color | Accent blue `#007AFF` (system `systemBlue`), 80% opacity |
| Ripple | Concentric ring on landing (1px stroke, fades from opaque to transparent over 400ms) |
| Cursor trail | No trail (clarity over aesthetics; single lerp path only) |

### Animation

| Property | Specification |
|----------|---|
| Easing | Ease-in-out cubic (40% in, 60% out, total duration target 200-400ms depending on distance) |
| Start point | Current cursor position (macOS will hide real cursor during lerp if needed) |
| End point | Target element center (from action_canonical.target_bbox) |
| Timing | **Lerp fires BEFORE action sends to translator**, so user sees the ghost cursor move to the target and the action appears to follow the cursor motion |
| Distance bucketing | <100px = 150ms lerp; 100-300px = 250ms; >300px = 350ms (human eye can resolve motion up to ~5px/ms before it looks unnatural) |

### Interaction Contract

- Ghost cursor never blocks real input (overlay ignores mouse/keyboard)
- If user moves real cursor during ghost lerp, ghost continues to planned target (no interruption)
- If target window moves/resizes between action selection and ghost fire, use last-known bbox (no live tracking — that adds latency)

## Element Highlight Box

### Visual Style

| Property | Specification |
|----------|---|
| Shape | Rounded rectangle (8px corner radius, matching iOS design language) |
| Border | 2px stroke in accent blue `#007AFF`, semi-transparent (60% opacity) |
| Fill | Transparent with accent color overlay at 10% opacity (light blue wash) |
| Label position | Top-left corner, 4px inset from border |
| Label font | SF Mono, 11px, weight 500 (monospace for tier badges) |
| Label background | Black with 80% opacity behind text (ensures readable on any background) |
| Z-order | Below ghost cursor (cursor is always topmost) |

### Lifecycle

- Highlight appears **BEFORE action fires** (pre-action visual confirmation)
- Highlight persists for 300ms after action lands (ripple duration)
- **Disabled during verify windows** (P11 — hide overlay during screenshot capture for verifier; verifier's SCContentFilter excludes the window anyway, but don't draw)
- Disappears when next action begins

## SwiftUI HUD (Heads-Up Display)

### Panel Layout

| Component | Specification |
|----------|---|
| Background | `.ultraThinMaterial` (system blur, 30% opacity baseline) |
| Corner radius | 12px (macOS standard) |
| Content inset | 16px (md spacing) |
| Width | 320px (fixed, right edge of screen or smart-snapped) |
| Height | Dynamic (min 60px for empty state, max ~400px for 8-action history) |
| Position default | Bottom-right corner, 32px from edge (lg spacing) |
| Draggable | YES — entire panel moves with mouse drag (custom trackpad gesture) |

### Content Structure

```
┌─────────────────────────────────┐
│  Session: 2026-05-01 09:34:22   │  ← timestamp (12px, weight 400)
│  Goal: [current task name]      │  ← 14px body, weight 500
├─────────────────────────────────┤
│ [T1] Click "Send" — VERIFIED    │  ← action 8 (oldest in view)
│ [C2]                            │
├─────────────────────────────────┤
│ [T2] Type "hello" — VERIFIED    │  ← action 7
│ [C5]                            │
├─────────────────────────────────┤
│ [T4] Locate button @ 320,180    │  ← action 6 (newest in view)
│ [C1]                            │
├─────────────────────────────────┤
│  ◀  ▶  [ opac: ═════════  ]     │  ← controls row
└─────────────────────────────────┘
```

### Action History Row Format

| Field | Specification |
|-------|---|
| Tier badge | T1-T5 in SF Mono, 11px, weight 600 in accent blue, left-aligned; 8px right-margin |
| Channel badge | C1-C5 in SF Mono, 10px, weight 500, gray `#666666`; right-aligned |
| Action text | 12px weight 400, monospace (action_canonical.action_type + short params) |
| Status icon | ✓ VERIFIED (green), ⚠ HEALING (orange), ✗ FAILED (red); 12px, right-aligned before channel badge |
| Divider | 1px line, 4px vertical padding, `#E5E5E5` (light gray) |
| Height | 44px (tap-friendly, matches macOS/iOS touch target) |

### Controls Row

| Control | Specification |
|---------|---|
| Prev button | SF Symbols `chevron.left`, 12px, clickable (cycles history back) |
| Next button | SF Symbols `chevron.right`, 12px, clickable (cycles history forward) |
| Opacity slider | Inline slider, range 0-100%, step 10%, default 70% |
| Snap checkbox | Icon `pin.fill` when engaged, toggles position snap-to-corner |
| Spacing | 8px between controls (sm token) |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Session header | `Session: YYYY-MM-DD HH:MM:SS` (ISO 8601 local time) |
| Goal prompt | `Goal: [user's task name from planner]` (1 line max, truncate with `…` if >40 chars) |
| Action entry | `[Tier] [action_type] [truncated_params] — [status]` |
| Status VERIFIED | `✓ VERIFIED` (green text) |
| Status HEALING | `⚠ HEALING (branch B2)` (orange, show active branch name) |
| Status FAILED | `✗ FAILED (Cognitive)` (red, show failure class) |

| Error state | `ERROR: [short problem] — See action_log.ndjson` (red text, 11px) |

**Tone:** Technical, factual, no natural language fluff. Every token visible contributes signal.

## 60fps H.265 Replay

### Recording Spec

| Property | Specification |
|----------|---|
| Source | ScreenCaptureKit (SCContentFilter excludes overlay window ID) |
| Codec | H.265 (HEVC), lossless profile if available; fall back to baseline for A15 Macs |
| Frame rate | 60fps (matches display refresh on Retina) |
| Resolution | Native screen resolution (no downscale unless >4K, then 1/2 scale) |
| Storage | `~/.cua/sessions/<id>/recording.mp4` (one file per session) |
| Metadata | NDJSON sidecar `~/.cua/sessions/<id>/recording_metadata.ndjson` with frame→step_idx mapping |

### Player UI

- Standard QuickTime / AVFoundation player (embed in Replay window)
- Timeline scrubber with step markers (overlaid ticks at every action boundary)
- Playback speed: 1x (default), 0.5x, 2x buttons
- Frame-by-frame: left/right arrow keys
- Audio: none (muted)

## 3D Timeline Visualization

### Axes

| Axis | Data | Scale |
|------|------|-------|
| X | Timestamp (session start = 0) | Linear, 0–session_duration_ms |
| Y | App/window identifier (bundleID + window title) | Categorical, one row per unique window touched |
| Z | Action depth (hierarchy in recovery / speculatively pre-executed branches) | Stacked layers: primary action at Z=0, branch 1 at Z=1, etc. |

### Rendering

- **Primary timeline:** white line tracing X-axis (time progression)
- **Action node:** colored dot at (X=timestamp, Y=app_row, Z=depth), diameter 8px, color = tier badge color (T1=blue, T2=cyan, T3=orange, T4=green, T5=red)
- **Branch divergence:** tree structure; primary path in bold white, alternate branches in light gray (30% opacity)
- **Counterfactual path:** dashed line showing "what if branch B had won"
- **Hover tooltip:** action_type, target_label, status (verified / healed / failed)
- **Click to scrub:** click any node to jump replay to that step

### Controls

| Control | Action |
|---------|--------|
| Zoom slider | X-axis zoom (0.1x to 10x, default 1.0x) |
| Y-axis auto-fit | Button to auto-scale Y (vertical fit-to-content) |
| Filter by app | Dropdown to show only one bundle_id's actions (all visible by default) |
| Playback | Play button syncs timeline marker with video frame |
| Branch view | Toggle to show only primary path vs. all branches |

## Counterfactual Replay

### Definition

When a step N has multiple recovery branches (B1, B2, B3, B4, B5) and a particular branch Bi verified, show what would have happened if branch Bj had won instead.

### Visual Representation

- **Primary timeline:** thick white line (what actually happened)
- **Counterfactual path:** dashed line in semi-transparent purple (what-if state)
- **Divergence point:** labeled at step N ("Branch divergence at step N: B2 won, showing B4 counterfactual")
- **Post-divergence states:** overlaid in semi-transparent overlay; user can toggle opacity

### Interaction

- Dropdown to select alternate branch: `Counterfactual: [Select] ▼ (showing B4, B2 won)`
- Toggle ON/OFF with Cmd+Shift+D (from hotkeys)
- Scrubbing through timeline updates counterfactual states in real-time

## Session Diff

### Compare Two Sessions

UX matches `git diff` / side-by-side compare:

```
Session A (left)          │  Session B (right)
2026-05-01 08:00         │  2026-05-01 09:30
─────────────────────────┼─────────────────────────
Goal: Compose email      │  Goal: Compose email
                         │  
[T1] Click inbox  ✓      │  [T1] Click inbox  ✓
[T2] Click compose ✓     │  [T2] Click compose ✓
[T1] Type to:... ✓       │  [T1] Type to:... ✓
[T3] Type subject ✓ (A)  │  [T3] Click subject ✓ (B)  ← DIFF: translator swap
                         │  [T1] Type subject ✓
[T2] Click send ✓        │  [T2] Click send ✓
                         │
HEAL-EVENTS:             │  
 • Step 4: T3→T1 swap    │  • Step 4: T3→T1 swap
   (reason: T3 timeout)  │  • Step 5: new step added
```

### Columns

| Column | Content |
|--------|---------|
| Left (A) | Action log from older session; action_log.ndjson events |
| Middle | Diff markers: `← SAME`, `← HEAL:` (changed), `← NEW`, `← REMOVED` |
| Right (B) | Action log from newer session |

### Diff Algorithm

- **Step alignment:** match on `(app, target_label, action_type)` tuple; if misaligned, show as removed (A) + added (B)
- **Translator swap:** highlight when same step uses different tier (T1→T2) — indicates previous session hit a translator timeout
- **Healing event:** when cassette shows a selector change (`old_locator` vs `new_locator`), show in HEAL-EVENTS block
- **Branch divergence:** if session A took recovery branch B1 and session B took B3, show in diff as competing branches

### Controls

| Control | Function |
|---------|----------|
| Session A picker | Dropdown to select first session from `~/.cua/sessions/` |
| Session B picker | Dropdown to select second session (defaults to latest) |
| Diff view | Button to toggle inline (current) vs. side-by-side layout |
| Focus filter | Dropdown to show all steps, or only diff steps (hide "same") |
| Export | Button to save diff as markdown to clipboard |

## Spacing Scale

Declared values (macOS design tokens, all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, inline padding in badges |
| sm | 8px | Small control spacing (buttons, sliders) |
| md | 16px | HUD content inset, standard element margin |
| lg | 24px | HUD margin from screen edge, section breaks |
| xl | 32px | Large layout gaps |

**Exceptions:** Ghost cursor circle (16px diameter) is not a spacing token — it's a visual dimension. Ripple stroke (1px) is intentionally sub-token for fineness.

### Palette

| Role | Value | Usage |
|------|-------|-------|
| Accent blue | `#007AFF` | Ghost cursor, highlight border, T1 badge |
| Accent cyan | `#32B4F9` | T2 badge (CDP translator) |
| Accent orange | `#FF9500` | T3 badge (AppleScript translator), warnings |
| Accent green | `#34C759` | T4 badge (Vision translator), success states |
| Accent red | `#FF3B30` | T5 badge (Pixel translator), errors, destructive |
| Gray text | `#666666` | Secondary copy (channel badges, metadata) |
| Gray background | `#F5F5F5` | Divider lines, subtle backgrounds |
| Black text | `#000000` | Primary copy, high contrast |
| White text | `#FFFFFF` | Text on dark backgrounds (label overlays) |

### Dark mode

Since this is a local dev tool on Akeil's Mac:
- **Support system dark mode** if available (query `NSApp.appearance`)
- **Default to light mode** for HUD panel (white blur material is more readable than dark at 30% opacity)
- Ghost cursor and highlights stay the same colors (accent colors are vibrant on both backgrounds)

### Accent Usage

Accent blue is **reserved for:**
- Ghost cursor circle and crosshair
- Element highlight border
- T1 (AX) translator badge
- Primary CTA buttons (none in HUD, but if added in future)

**Never** use accent color for all interactive elements (violates 10% rule — accent is sparse).

## Interaction Flows

### User Toggles HUD Visibility

1. User presses Cmd+Shift+V
2. Global hotkey handler fires (AppDelegate)
3. Visualizer process receives event over unix socket
4. NSPanel `isHidden` toggles (animation: fade 200ms)
5. HUD slides in/out (no stutter — pre-rendered during fade)

### User Scrubs Timeline

1. User clicks node on 3D timeline
2. Timeline sends `{event: "scrub_to_step", step_idx: N}` to replay engine
3. Video player seeks to frame corresponding to step N (from metadata.ndjson)
4. Action history scrolls to show step N as newest entry in HUD
5. Counterfactual (if active) re-renders for post-divergence states starting at N+1

### User Compares Two Sessions

1. User clicks "Session Diff" in timeline UI
2. Modal opens with two session dropdowns
3. User selects Session A (older) and Session B (newer)
4. Diff view renders side-by-side action logs
5. Aligned steps are matched on `(app, target, action_type)` tuple
6. Misaligned steps show as removed/added
7. User can toggle "show diffs only" to hide matching steps
