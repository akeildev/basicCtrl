# Phase 03 Plan 04: 5 Recovery Branches B1-B5 (Wave 2)

**Implement 5 parallel recovery branches: B1 rescroll+retry, B2 OCR regrounding, B3/B4 stubs (Phase 4), B5 AppleScript fallback. Each reuses Phase 2's translators/channels/idempotency; unit tests cover success/failure paths.**

## Objective

Implement the 5 recovery branches that execute when verification fails. Each branch attempts a different recovery strategy. Branches use the existing Phase 2 translator + channel infrastructure; B3/B4 are stubs in Phase 3 (full Opus planner + world model come in Phase 4). All branches respect idempotency tokens to prevent double-fires (T-3-05).

Purpose: Provide concrete recovery paths for each failure class (routed by Plan 03-02 classifier); enable first-verified-branch pattern (first branch that passes verifier wins, others cancelled).

Output: `basicctrl/recovery/branches/` package with 5 branch implementations + 30+ unit tests covering mocked success/failure scenarios.

| T-3-05 | Spoofing | recovery-induced double-action | mitigate | All branches call IdempotencyTokenStore.try_claim before any channel.fire; return None if claim already owned |

</threat_model>
