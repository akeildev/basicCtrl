# Phase 03 Plan 02: Failure Classifier + HealEvent Models (Wave 1)

**Define 6-class failure enum with routing dispatch table (D-01); Pydantic HealEvent model for stable-tier-gated heal write-back (D-14); unit tests cover all routes.**

## Objective

Implement failure classification logic that feeds the 5-branch recovery system. Classify verifier failures into 6 failure classes (PERCEPTUAL / COGNITIVE / ACTUATION / ENVIRONMENTAL / RESOURCE / LOOP), each mapped to specific recovery branches. Also define HealEvent Pydantic model used by all recovery branches.

Purpose: Unblock Wave 2 recovery branches (they depend on FailureClass for routing); emit auditable heal events that defeat silent-mask regression trap (P20, T-3-01).

Output: `cua_overlay/recovery/classifier.py` + `heal_event.py` with full unit coverage.

  <name>Task 1: Implement FailureClass classifier with 6-class enum and dispatch table (D-01)</name>
  <files>cua_overlay/recovery/classifier.py</files>
  <action>Create classifier.py implementing:

  1. **FailureClass Pydantic enum** with 6 variants:
     - `PERCEPTUAL` — screen changed unexpectedly (wrong element clicked, vision drift)
     - `COGNITIVE` — action didn't make logical sense (wrong step in flow)
     - `ACTUATION` — OS rejected action (kAXErrorCannotComplete, target disabled, window hidden)
     - `ENVIRONMENTAL` — external state changed (app crashed, network down, TCC denied)
     - `RESOURCE` — out of capacity (AX main-thread saturation, memory pressure)
     - `LOOP` — repeated failures on same target (circuit breaker candidate)

  2. **FailureCtx TypedDict** with fields:
     - `bundle_id: str`
     - `target_key: str` (composite key from Phase 1)
     - `hoare_post: HoarePost` (from RaceOrchestrator result)
     - `confidence: float` (verifier confidence 0-1)
     - `last_error: str` (error message from last verifier/translator)
     - `previous_failures_count: int` (for circuit breaker logic)

  3. **FailureClassifier.classify(ctx: FailureCtx) -> FailureClass** method:
     - Decision tree based on confidence + error patterns:
       - confidence < 0.10 → PERCEPTUAL
       - confidence 0.10-0.30 + "kAXErrorCannotComplete" → ACTUATION
       - confidence 0.30-0.50 + "cdp ws closed" → ENVIRONMENTAL
       - confidence 0.30-0.50 + "timed out" → RESOURCE
       - confidence 0.50-0.70 + "unexpected state" → COGNITIVE
       - confidence > 0.70 + previous_failures_count >= 3 → LOOP
       - default: PERCEPTUAL
     - Return confidence tuple: (class, confidence_pct) where confidence_pct reflects decisiveness of routing

  4. **Branch dispatch table** (module-level dict):
     ```
     FAILURE_CLASS_TO_BRANCHES = {
         FailureClass.PERCEPTUAL: [B1_RESCROLL, B2_OCR_REGROUND, B4_PLANNER],
         FailureClass.COGNITIVE: [B3_WORLD_REPLAN, B4_PLANNER],
         FailureClass.ACTUATION: [B1_RESCROLL, B2_OCR_REGROUND, B5_APPLESCRIPT],
         FailureClass.ENVIRONMENTAL: [B5_APPLESCRIPT, B4_PLANNER],
         FailureClass.RESOURCE: [B2_OCR_REGROUND, B1_RESCROLL],
         FailureClass.LOOP: [B5_APPLESCRIPT],  # Last resort
     }
     ```
     (B3/B4 will emit stubs in Phase 3; full implementation Phase 4)

  Include module docstring and type hints per Phase 1/2 pattern.

  - `frozen=True` (immutable, safe to use as event payload)

  2. **is_stable_tier() classmethod** — returns True if locator_tier in {AXIdentifier, AXLabel, AXTitle, AXRoleDescription} (D-20). Vision and Coordinate return False (session-only, never write back).

  3. **serialize_for_ndjson() method** — returns dict suitable for NDJSON (ts as ISO string, rest as-is).
