# Phase 03 Plan 05: RecoveryOrchestrator + Bounded Cycles (Wave 3)

**Wire all 5 recovery branches into parallel orchestrator using anyio.create_task_group + first_verified pattern (reuse Phase 2's race_first_complete). Enforce max 2 cycles, circuit breaker reordering, heal-rate budget; emit recovery_log events for RL training.**

## Objective

Orchestrate the 5 recovery branches in parallel after verification failure. Use the Phase 2 racing pattern (anyio task group, cancel_scope.cancel on first winner) to execute branches concurrently. Track recovery cycles (max 2, then escalate); consult circuit breaker before branching; track heal-rate budget to prevent silent regression masking.

Purpose: Execute the full recovery loop: classify failure → branch fanout → first-verified wins → record outcomes → check circuit breaker → emit events. Bounded by cycles and heal-rate to prevent infinite loops and cost explosions (P26, T-3-03).

Output: `basicctrl/recovery/orchestrator.py` with full recovery orchestration logic + 15+ unit tests.

     - `branches_list: list[RecoveryBranch]` — [B1, B2, B3, B4, B5]

  2. **RecoveryOrchestrator.attempt(failure_ctx: FailureCtx) -> tuple[Optional[ChannelOutcome], List[dict]]**:
     - Returns: (winning_outcome, recovery_log_events) where recovery_log_events is list of structured dicts for RL training
     
     Core loop (D-11 bounded cycles):
     ```
     cycle = 0
     while cycle < max_cycles:
         cycle += 1
         
         # D-12/D-13: Circuit breaker check
         if circuit_breaker.is_tripped(failure_ctx.bundle_id, failure_ctx.target_key):
             emit event: {"event": "recovery_skipped_breaker_tripped", ...}
             escalate_to_user()
             return (None, recovery_log_events)
         
         # D-16: Heal-rate budget check
         if heal_rate > 0.05:
             emit event: {"event": "recovery_skipped_heal_budget_exceeded", ...}
             escalate_to_user()
             return (None, recovery_log_events)
         
         # D-01/D-02: Classify failure
         failure_class = classifier.classify(failure_ctx)
         branches_to_try = FAILURE_CLASS_TO_BRANCHES[failure_class]
         
         # D-09: Fan out 5 branches in parallel
         winning_outcome, all_outcomes = await race_first_complete(branches_to_try, failure_ctx)
         
         if winning_outcome:
             # D-03: Log success to recovery_log
             emit event: {"event": "recovery_succeeded", "cycle": cycle, "winning_branch": ..., ...}
             return (winning_outcome, recovery_log_events)
         
         # D-10: Log failed branches to RL training buffer
         for branch, outcome in all_outcomes:
             emit event: {"event": "recovery_branch_failed", "branch": branch.name, "reason": outcome.error, ...}
         
         # D-12: Record failure for circuit breaker
         circuit_breaker.record_failure(failure_ctx.bundle_id, failure_ctx.target_key, app_profile)
         
         # Prepare for next cycle (if any)
         failure_ctx.previous_failures_count += 1
     
     # D-11: Escalate after max cycles
     emit event: {"event": "recovery_exhausted", "cycles_tried": max_cycles, "branches_attempted": ..., ...}
     escalate_to_user()
     return (None, recovery_log_events)
     ```

  3. **async def race_first_complete(branches, ctx) -> tuple[Optional[ChannelOutcome], List[tuple]]**:
     - Same pattern as Phase 2's race_first_complete
     - Use anyio.create_task_group() + cancel_scope.cancel() when first branch succeeds
     - Return (winning_outcome, [(branch1, outcome1), (branch2, outcome2), ...])
     - Catching CancelledError is OK (losers are cancelled)

  4. **def escalate_to_user() helper**:
     - Build structured event: `{"event": "recovery_escalated", "action_id": ..., "target_key": ..., "last_error": ..., "branches_tried": [...], "suggested_action": "..."}`
     - Emit to SessionWriter
     - In Phase 3, this just logs; Phase 5 will surface in HUD

  5. **Heal-rate budget tracking**:
     - Track `_heal_event_count` (incremented by HealEvent emitters in Phase 4 write-back)
     - Track `_total_actions` (incremented by RaceOrchestrator after each action)
     - Compute ratio: `_heal_event_count / max(1, _total_actions)` >= 0.05 triggers pause
