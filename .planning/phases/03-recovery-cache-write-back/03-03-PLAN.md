# Phase 03 Plan 03: Circuit Breaker (Wave 1)

**Implement per-(bundle_id, target_key) failure counter; trip after 3 consecutive failures; reorder translator priority; emit structured event (D-12, D-13).**

## Objective

Build circuit breaker that prevents cascading recovery attempts on broken targets. After 3 consecutive failures on the same target within a 60s window, trip the breaker: reorder the AppProfile's translator_priority list (move primary to tail, promote next), emit a structured event, and prevent auto-heal for 60s. Supports manual reset via MCP tool.

Purpose: Mitigate P26 (5-branch recovery cost explosion); defeat silent-mask pattern (failing forever with no signal to user).

Output: `cua_overlay/recovery/circuit_breaker.py` with full state management + unit tests.

     - `__init__(session_writer: SessionWriter)` — store reference to SessionWriter for event emission
     - `_state: dict[str, BreakState]` — in-memory async.Lock-guarded dict mapping `f"{bundle_id}:{target_key}"` to BreakState

  3. **CircuitBreaker.record_failure(bundle_id: str, target_key: str, app_profile: AppProfile) -> bool**:
     - Record one failure on the target.
     - If within 60s window since first failure: increment failure_count.
     - If count >= 3: trip the breaker, reorder app_profile.translator_priority (move index 0 to end, shift others left), emit circuit_breaker_tripped event, return True (breaker just tripped).
     - If count < 3: return False (still collecting failures).
     - If 60s elapsed since first failure: reset count to 1, return False.

  4. **CircuitBreaker.is_tripped(bundle_id: str, target_key: str) -> bool**:
     - Return True if breaker tripped and <60s since trip.

  5. **CircuitBreaker.reset(bundle_id: str, target_key: str)**:
     - Manually clear state (MCP tool D-25 will call this).

  6. **CircuitBreaker._emit_trip_event(bundle_id, target_key, failure_count)**:
     - Call `session_writer.append_action_log({"event": "circuit_breaker_tripped", "bundle_id": bundle_id, "target_key": target_key, "failure_count": failure_count, "ts": datetime.utcnow().isoformat()})`

  Use asyncio.Lock (same pattern as Phase 2 IdempotencyTokenStore).
