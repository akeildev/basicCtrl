1. Create `Grounder` class:
   ```python
   class Grounder:
       async def ground_ui_tars(self, 
                               screenshot: bytes,
                               instruction: str) -> tuple[tuple[float, float, float, float], float]:
           """
           UI-TARS-1.5-7B via mlx-vlm 0.4.4.
           Output: (x, y, w, h) bbox + confidence [0, 1]
           Per D-04, P4: apply sanity_gate BEFORE returning.
           """
       
       async def sanity_gate(self,
                            bbox: tuple[float, float, float, float],
                            screenshot_w: int,
                            screenshot_h: int,
                            expected_element_hash: str | None = None) -> bool:
           """
           P4 mitigation: reject if |x - W/2| < 10 AND |y - H/2| < 10 (screen center).
           UNLESS pixel-hash of expected element confirms center IS the target (edge case: center button).
           Returns True if bbox passes gate, False to trigger fallback.
           """
       
       async def fallback_to_uitag(self,
                                  screenshot: bytes,
                                  instruction: str) -> tuple[float, float, float, float]:
           """
           D-05 primary grounder: uitag SoM (Apple Vision + YOLO11 MLX).
           Call t4_vision.ground_with_uitag() or equivalent.
           Differential grounding gate: IoU > 0.5 required vs UI-TARS bboxes.
           """
   ```

2. Implementation details:
   - Load UI-TARS-1.5-7B 4-bit MLX model from mlx-community (first call triggers ~4.5GB HF download)
   - Convert screenshot bytes → PIL Image → MLX input tensors
   - Run inference, extract bbox + confidence
   - **Sanity gate logic**: `center_x = screenshot_w / 2; center_y = screenshot_h / 2; if abs(bbox.x - center_x) < 10 and abs(bbox.y - center_y) < 10: check_expected_element_hash() or fallback_to_uitag()`
   - If sanity gate fails → call fallback_to_uitag() (D-05)
   - D-05 differential grounding: compute IoU(UI_TARS_bbox, uitag_bbox); if IoU < 0.5, use T4 OCR fallback instead
