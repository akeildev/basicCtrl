# Phase 6 — Private SPIs + Durability Hardening Demo

**Goal:** Verify all 8 SPI implementations work correctly. Demonstrate graceful degradation on default Mac (SIP on). Validate durability (kill -9 resume).

**Automated test suite baseline:** All `pytest tests/test_spi_*.py tests/test_durability.py -x` passing.

---

## Manual Demo Cases

### Case 1: SkyLight Background Event (SPI-01)

**Scenario:** C1 SkyLight channel fires background event with no cursor warp.

**Setup:**
1. Open Slack
2. Ensure Slack window is occluded (click another window)
3. Start Phase 6 agent in test mode: `python -m cua_overlay.demo --phase 6`

**Action:**
```bash
python -m cua_overlay.demo --action click_skylight \
  --target "com.tinyspeck.macslack" \
  --coordinate "500,300"
```

**Expected Output:**
- Agent log shows: `[INFO] c1_spi_fire_success via=SkyLight`
- Slack receives click WITHOUT cursor warping to coordinate
- Slack message field focused (if target was message box)
- No visual cursor movement on screen

**Pass Criteria:** SkyLight event delivered silently (no cursor visible).

---

### Case 2: AX Remote Notifications (SPI-02)

**Scenario:** Background Slack automation via occluded AX trees.

**Setup:**
1. Open Slack + VS Code (or Discord)
2. Occlude Slack (click VS Code)
3. Agent targets a Slack message element via T1 AX

**Action:**
```bash
python -m cua_overlay.demo --action ax_remote_subscribe \
  --target "com.tinyspeck.macslack" \
  --notification "kAXValueChanged"
```

**Expected Output:**
- Agent log: `[INFO] ax_remote_subscription_created target_pid=<pid>`
- Slack AX tree remains accessible even though occluded
- Type message in occluded Slack → kAXValueChanged fires (via AXObserver push)

**Pass Criteria:** Background AX notifications work while app occluded.

---

### Case 3: Tier-B SPI Graceful Degradation (SPI-04, SPI-05)

**Scenario:** ES + DTrace unavailable on default Mac (SIP on). Agent gracefully skips.

**Setup:**
```bash
csrutil status  # Confirm "System Integrity Protection status: enabled."
```

**Action:**
```bash
python -m cua_overlay.demo --action probe_endpoint_security
python -m cua_overlay.demo --action probe_dtrace
```

**Expected Output:**
- Agent log: `[INFO] endpoint_security_unavailable_on_default_mac`
- Agent log: `[INFO] dtrace_unavailable_on_default_mac`
- Both probes return False (capabilities.endpoint_security_available = False)
- Agent continues to run; no crash

**Pass Criteria:** Unavailable SPIs logged clearly; agent degrades gracefully.

---

### Case 4: IMU Sensor Detection (SPI-08)

**Scenario:** M-series: IMU detected. Intel: gracefully unavailable.

**Action:**
```bash
python -m cua_overlay.demo --action probe_imu
ioreg -r -d 1 -c AppleSPUHIDDevice  # Verify sensor presence
```

**Expected Output (M-series):**
- Agent log: `[INFO] imu_service_discovered`
- ioreg output contains AppleSPUHIDDevice service

**Expected Output (Intel):**
- Agent log: `[WARNING] imu_discovery_failed` or sensor not found
- Agent continues; feature marked unavailable

**Pass Criteria:** Correct detection per hardware. No crashes.

---

### Case 5: Durability — Kill -9 Resume (PERSIST-01)

**Scenario:** Crash mid-action; restart resumes from last verified checkpoint.

**Setup:**
1. Start durable session: `python -m cua_overlay.main --persistent`
2. Postgres running: `brew services status postgresql@16`

**Action (simulate crash):**
```bash
# Terminal 1: Run agent
python -m cua_overlay.demo --action long_action_sequence

# Terminal 2 (while Terminal 1 running): Crash process
ps aux | grep cua_overlay
kill -9 <pid>

# Terminal 1: Restart same session
python -m cua_overlay.main --resume-session <session_id>
```

**Expected Output:**
- On crash: Agent process terminated (no graceful shutdown)
- On restart: Agent logs: `[INFO] resume_from_checkpoint session_id=<id> step_idx=5`
- Agent resumes from action 6 (skipping 1-5, which were verified)
- State graph restored from checkpoint (hoare_post from step 5)

**Pass Criteria:** Agent resumes from last verified step (not step 0).

---

**Scenario (if spike RED):** Graceful unavailability; fallback to T1 AX.

**Expected Output (GREEN):**
- Agent log: `[INFO] dyld_inject_bridge_loaded`
- Dylib loads: `[INFO] DYLDInject.dylib loaded in Electron renderer`
- Renderer introspection works

**Expected Output (RED):**
- Agent log: `[WARNING] dyld_inject_unavailable; fallback to T1 AX`
- Agent uses T1 AX for Electron (lossy but functional)

**Pass Criteria:** Works if spike GREEN, or gracefully unavailable if RED.
